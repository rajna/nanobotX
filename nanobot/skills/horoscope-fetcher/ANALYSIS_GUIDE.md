# 星盘深度分析使用指南

## 概述

`horoscope-fetcher` skill 现在支持 AI 驱动的星盘深度分析功能。该功能使用大语言模型（LLM）基于图结构框架，生成优雅、富有哲思和洞见的本命盘解读。

## 功能特点

### 图结构分析框架

分析过程采用图结构作为分析框架，包含：

- **节点**：星体、星座、宫位、相位、逆行等元素的组合
- **边**：节点之间的关系，包括：
  - 相位的和谐与挑战
  - 星座的特质、共性与对立
  - 星体的特质、共性与对立
  - 星座和星体的对应关系

### 分析流程

1. **构建图结构**：根据星盘信息，分析推理生成图结构
2. **遍历分析**：按照图结构，从某个节点出发，经过一系列节点进行分析
3. **文本调整**：将节点和边的代号转化为流畅的占星语言
4. **最终交付**：检查并调整成最终交付文本

## 使用方法

### 前置要求

在使用深度分析功能之前，需要设置 OpenAI API Key：

```bash
export OPENAI_API_KEY="your-api-key-here"
```

### 基本用法

```bash
python3 scripts/horoscope_fetcher.py analyze_chart \
  --time "1990-01-01 12:00:00" \
  --latitude 39.9042 \
  --longitude 116.4074
```

### 使用地址

```bash
python3 scripts/horoscope_fetcher.py analyze_chart \
  --time "1990-01-01 12:00:00" \
  --address "北京"
```

### 指定模型

```bash
python3 scripts/horoscope_fetcher.py analyze_chart \
  --time "1990-01-01 12:00:00" \
  --latitude 39.9042 \
  --longitude 116.4074 \
  --model "gpt-4o"
```

## 参数说明

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `--time` | string | 是 | 出生时间（格式：YYYY-MM-DD HH:MM:SS） |
| `--address` | string | 否* | 出生地点（城市名称） |
| `--latitude` | float | 否* | 纬度（十进制） |
| `--longitude` | float | 否* | 经度（十进制） |
| `--timezone` | string | 否 | 时区（默认：GMT） |
| `--model` | string | 否 | 使用的 LLM 模型（默认：gpt-4o-mini） |

* 必须提供 `--address` 或 `--latitude/--longitude` 其中之一

## 输出示例

分析结果包含以下部分：

1. **图结构**：以树状图形式展示星盘的图结构
2. **深度分析**：基于图结构的详细解读，包括：
   - 三大巨头（太阳、月亮、上升）
   - 宫位和半球分析
   - 焦点行星群
   - 关键单星和交点

## 注意事项

1. **API Key**：必须设置 `OPENAI_API_KEY` 环境变量
2. **网络连接**：需要能够访问 OpenAI API
3. **分析时间**：深度分析可能需要 30-60 秒，请耐心等待
4. **模型选择**：
   - `gpt-4o-mini`：速度快，成本较低（推荐）
   - `gpt-4o`：分析更深入，成本较高
   - `gpt-3.5-turbo`：速度快，但分析深度较浅

## 故障排除

### 错误：未找到 OPENAI_API_KEY 环境变量

**解决方案**：设置环境变量
```bash
export OPENAI_API_KEY="your-api-key-here"
```

### 错误：调用大模型失败

**可能原因**：
- 网络连接问题
- API Key 无效
- API 配额已用完

**解决方案**：
- 检查网络连接
- 验证 API Key 是否正确
- 检查 OpenAI 账户配额
